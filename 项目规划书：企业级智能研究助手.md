### 项目计划书

#### 项目名称：AI Agent 项目

---

### 1. 项目背景与目标

#### 背景：

随着人工智能技术的迅速发展，AI 应用在各类行业中的应用逐渐深入。对于 AI 应用开发相关岗位的面试，展示对现代 AI 技术的理解和实际应用能力，能够显著提高竞争力。本项目旨在开发一个 AI Agent，用于在个人知识库中（例如课件、作业、撰写的内容）进行智能搜索和问答。该系统将帮助用户通过上传文件构建知识图谱和向量库，并通过 AI 技术提供高效的检索服务。

#### 目标：

- 开发一个 AI Agent，提供用户自有知识库的智能搜索和问答服务。
- 实现前后端分离，具备友好的用户界面和交互体验。
- 熟练应用并展示 AI 领域的核心技术，涵盖 LangChain/LangGraph、RAG、LLM、用户鉴权、容器化等技术。

---

### 2. 技术栈

#### 核心技术：

1. **LangChain/LangGraph**：构建和管理语言链与图谱结构，支持基于文档的智能问答。
2. **RAG/GraphRAG**：结合检索与生成技术（Retrieval-Augmented Generation），增强问答的准确性和上下文关联。
3. **MCP**：管理和优化多模型的组合，以提高推理和决策过程。
4. **Agent**：构建智能代理，能够执行任务并根据需求进行自我学习与调整。
5. **LLM（大语言模型）**：如 GPT-4 等大语言模型，应用于自然语言理解与生成。
6. **Python**：后端主要编程语言，负责实现推理、数据处理、模型调用等功能。
7. **FastAPI**：用于构建后端 API，提供高性能的接口服务。
8. **React**：前端框架，负责构建用户界面和用户交互功能。
9. **MongoDB**：数据库管理系统，用于存储用户数据，如账户信息、上传的文件目录等。
10. **JWT**：用户鉴权认证，用于管理用户的登录、注册和权限。
11. **Docker**：容器化技术，简化开发、测试、部署流程，确保应用的可移植性。
12. **Github**：用于版本控制和代码托管，方便团队协作和代码管理。
13. **Prompt 工程**：设计并优化与 AI 模型的交互，创建精准的提示信息，以提高系统的响应质量。
14. **Context 工程**：实现上下文管理，确保 AI 模型在处理用户查询时能够保持连贯的上下文，提高答复的准确性和相关性。

---

### 3. 功能需求

#### 前端：

1. **用户注册与登录**：提供注册和登录功能，支持 JWT 认证以确保用户数据的安全性。
2. **文件目录管理**：允许用户上传文件夹，支持对上传的课件、作业等文件进行管理。
3. **AI 对话界面**：提供一个交互式对话界面，用户可以向 AI 提问，获取相关知识库中的信息。
4. **文件夹选择与知识图谱构建**：用户上传文件夹时，系统自动提取内容并构建知识图谱和向量库。

#### 后端：

1. **用户数据管理**：使用 MongoDB 存储用户信息，包括账户数据、上传的文件目录等。
2. **文件上传与存储**：处理用户上传的文件，提取文本信息，存入系统中以供后续处理。
3. **知识图谱与向量化数据库**：使用 Chroma 存储和检索文件的嵌入向量，结合 LangChain 进行推理。
4. **问答推理**：结合 RAG 技术，为用户提供基于其上传文件的智能问答。
5. **多模型组合（MCP）**：支持多个模型的并行调用，提高系统的推理能力。
6. **JWT 用户认证与权限管理**：通过 JWT 实现用户认证与权限控制，保护用户数据安全。
7. **Prompt 工程**：设计和优化与 AI 模型交互的提示信息，确保模型能够理解用户的请求并给出准确答复。
8. **Context 工程**：实现上下文管理，保证问答的连贯性，并能根据用户的历史对话调整回答内容。

---

### 4. 系统架构设计

1. **前端架构**：

   - 使用 React 构建前端页面，确保用户界面的友好性与交互体验。
   - 提供用户注册、登录、文件管理、AI 问答等功能模块。
   - 通过 RESTful API 与后端进行数据交互，确保前后端分离。

2. **后端架构**：

   - 使用 FastAPI 构建后端服务，提供高效的 RESTful API 接口。
   - 实现用户认证、文件上传、问答推理等功能。
   - 使用 MongoDB 存储用户的相关数据，确保数据的安全性和高效访问。
   - 使用 Chroma 进行知识库的向量化存储与检索，结合 RAG 技术提供智能问答。
   - 在 Prompt 工程中，设计和优化用户查询的提示信息。
   - 在 Context 工程中，管理用户的对话上下文，确保问答的连贯性。

3. **数据库架构**：

   - **MongoDB**：负责存储用户数据、文件信息以及构建的知识图谱。
   - **Redis**：用于缓存常用查询，提升系统性能。
   - **Chroma**：向量数据库，存储文档的向量信息，支持快速检索。

4. **部署架构**：

   - 使用 Docker 容器化部署应用，确保开发、测试、生产环境的一致性。
   - 使用 Github 进行版本控制，确保代码的高效管理与协作。

### 7. 总结

本项目将帮助展示对 AI 技术的深入理解与实际操作能力，结合前后端开发与 AI 技术，开发一个智能问答系统。通过实现前后端分离、用户认证、智能推理等功能，确保项目的完整性与实用性，同时也能够在面试中展示全面的能力。
